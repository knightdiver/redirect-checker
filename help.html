FM Dev's Redirect Checker - User Guide

Overview

The Redirect Checker is a tool used to test if website redirects are properly configured. By uploading a list of URLs, the tool verifies whether they redirect to the expected destination.

Supported File Types

The tool accepts the following file formats:

  • .txt / .conf – Nginx rewrite rule files
  • .csv – Comma-separated values with old and expected URLs

Accepted File Formats

1. Nginx Rewrite Rules

If using an Nginx rewrite configuration file, it should contain lines like this:

rewrite ^/old-page$ https://example.com/new-page permanent;
rewrite ^/blog/post$ /new-post permanent;

The tool will extract the patterns and validate their redirects.

NOTE: — Currently, this utlity does not process Nginx 301/302 directives

2. CSV Format

CSV files must contain at least two columns:

  • First column: Old URL (can contain the full URL or just the path)
  • Second column: Expected Redirect URL

Example CSV content:

http://example.com/old-page,https://example.com/new-page
https://olddomain.com/old-product,https://newdomain.com/new-product
/old-route,https://newdomain.com/new-route

The tool will strip the domain from the first column if it's present.

Redirect Statuses

After processing, each redirect is assigned one of the following statuses:

  • ✅ Success – The redirect matches the expected URL.
  • ❓ Domain Mismatch – Different domain or no domain in expected.
  • ⚠️ Mismatch – The redirect does not match the expected destination.
  • ❌ Failed – No redirect was found for the given URL.

Explanation of Input Fields

  • Old Domain: Enter the domain of the old URLs (e.g., www.example.com).
  • Protocol: Choose whether the URLs should be checked under HTTP or HTTPS.
  • Upload Redirect File: Select a .conf, .txt, or .csv file.

Once all inputs are set, click "Test Redirects" to begin checking.
